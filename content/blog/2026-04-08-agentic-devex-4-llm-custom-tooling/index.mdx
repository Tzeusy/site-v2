## Personally maintained skills

### Diagramming: Excalidraw

<ExcalidrawDiagram light="amenities-light.svg" dark="amenities-dark.svg" alt="I've found LLM-generated Excalidraw diagrams excellent ways to improve my personal mental model of the project's design" />

(This was initially forked off this [excalidraw-diagram-skill](https://github.com/coleam00/excalidraw-diagram-skill) repository, and I further tuned it for themes and image export formats)

As code generation gets cheaper, my understanding of the current state of the system is the new cognitive bottleneck on project development. I have found that a good architecture diagram, topology map, or flow sketch often collapses pages of prose into something an agent can reason about in one glance. This is especially true for project-shape work, where much of the challenge is not implementation difficulty but preserving a coherent mental model across doctrine, contracts, requirements, and topology.

I have loved how this skill has allowed me to generate diagrams that intuitively explain how the infrastructure of my code works; this has short-circuited a lot of logical challenges I've had in investigating and resolving of issues.

### Blogpost Editing

Skills: [blogpost-editor](https://github.com/Tzeusy/ai-bootstrap/tree/main/skills/personal/blogpost-editor) and [blogpost-formatter](https://github.com/Tzeusy/ai-bootstrap/tree/main/skills/personal/blogpost-formatter)

While I try to write most my my blogposts with my own hand, using LLMs as an *editor* has been invaluable for my writing. Many times it has identified outdated or inaccurate claims, allowing me to tune these; prior to LLMs existing, I wouldn't have been able to resolve them without spending orders of magnitudes more time fact checking every single part of my posts. 

It is admittedly quite sycophantic, which frustrates me a little; GPT-5.4 frames posts which seem perfectly ordinary to me as 'excellent and consistent humor', which I *know* just sounds too good to be true for a first-draft post.
